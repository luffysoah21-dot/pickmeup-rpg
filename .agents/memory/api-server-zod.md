---
name: API server zod imports
description: How to use zod validation in Express route handlers in this project
---

API server routes must NOT import from `zod` or `zod/v4` directly — those packages aren't listed as a direct dependency of `@workspace/api-server`.

Instead, use schemas exported from `@workspace/api-zod` (auto-generated by orval from openapi.yaml):

```typescript
import { PerformSummonBody, RecordBattleBody } from "@workspace/api-zod";
const parsed = PerformSummonBody.safeParse(req.body);
```

**Why:** The api-server package.json only lists `@workspace/api-zod` as a dependency, not `zod` itself. Importing bare `zod` causes TS2307 at typecheck time.

**How to apply:** Any time you add a new route that needs request body validation, add the new schema to openapi.yaml first, run codegen, then import the generated schema from `@workspace/api-zod`.
